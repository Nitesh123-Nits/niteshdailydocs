    Inner Class

    üß© First: What an Inner Class Really Is¶

    An inner class (or nested class) is a class defined within another class.
    In Java, there are 4 types:

    1. Static nested class

    2. Non-static inner class

    3. Local inner class (inside a method)

    4. Anonymous inner class (inline class without a name)

    üß† Why Java Has Inner Classes (Design Rationale)¶

    Java added inner classes to support tight logical grouping and encapsulation ‚Äî when one class is only relevant to another and doesn‚Äôt make sense elsewhere.

    Think of it like:

    "I‚Äôm defining this class only because it helps my outer class internally ‚Äî it‚Äôs not a reusable, public entity."

    ‚öôÔ∏è Practical Advantages (Over Outer Classes)¶

    1. Tighter Encapsulation¶

    Inner classes can access private members of the outer class directly (even if private).
    That means you can keep your outer class‚Äôs internal structure hidden from others without exposing unnecessary getters/setters.

    ‚úÖ Example:

    public class LinkedList {
    private Node head;

    private class Node {
        int data;
        Node next;

        Node(int data) {
            this.data = data;
        }
    }

    public void add(int data) {
        Node newNode = new Node(data);
        newNode.next = head;
        head = newNode;
    }
}

    Here, Node is only meaningful inside LinkedList, so defining it inside keeps your model clean and cohesive.

    If you made Node an outer class, it would be unnecessarily visible and pollute the namespace.

    2. Improved Readability and Cohesion¶

    When a helper class exists solely to support its outer class, placing it inside shows that relationship explicitly.

    It‚Äôs like saying ‚Äî ‚Äúthis is not a separate component; it‚Äôs part of the internal mechanism‚Äù.

    3. Avoids Name Pollution¶

    In large systems, you often have many small helper classes.
    Making them outer classes increases top-level class count and pollutes packages.
    Inner classes help keep those local to where they belong.

    4. Anonymous Inner Classes for Callbacks or One-Time Use¶

    Used extensively before lambdas came along ‚Äî for listeners, comparators, etc.

    ‚úÖ Example:

    button.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        System.out.println("Button clicked!");
    }
});

    Saves you from creating a separate named class for a one-off implementation.

    5. Static Nested Classes Reduce Memory Footprint¶

    If the inner class doesn‚Äôt need access to outer class members, making it static avoids holding a reference to the outer object.

    ‚úÖ Example:

    public class Outer {
    static class Helper {
        void help() { System.out.println("Helping..."); }
    }
}

    Here, Helper is logically tied to Outer, but independent in lifecycle ‚Äî so we avoid creating unnecessary outer references.

    ‚öñÔ∏è When to Use Inner Classes (Best Practices)¶

    Situation Recommendation
    Class is only meaningful to its enclosing class ‚úÖ Use non-static inner class
    Class is logically related but independent ‚úÖ Use static nested class
    Class is a small one-time use ‚úÖ Use anonymous or local class
    Class is reusable in multiple contexts üö´ Make it a top-level outer class

    üß© In Real-World Use¶

    • Collections Framework uses many inner classes (e.g., HashMap.Node, ArrayList.Itr).

    • Builders, iterators, event handlers, and state machines often use them.

    • They help structure code where object relationships are tight but localized.

    üí° In Short¶

    Outer class:

    Represents a standalone concept or entity.

    Inner class:

    Represents something that exists only because the outer class exists ‚Äî tightly coupled behavior or state.

    ‚úÖ Refined Understanding¶

    ‚úîÔ∏è If an anonymous class is created inside a static context (like a static block, static method, or static nested class),
    it‚Äôs an anonymous class, not an inner class.

    ‚úîÔ∏è If it‚Äôs created inside a non-static context (like an instance method, instance initializer, or inner class),
    it‚Äôs an anonymous inner class.

    ‚öôÔ∏è Why?¶

    The difference lies in whether an enclosing instance exists.

    üîπ Static context¶

    No instance of the outer class exists ‚Äî only the class itself is loaded.
    So, the anonymous class has no reference to any outer object ‚Üí it‚Äôs not an inner class.

    üîπ Non-static context¶

    There is an active instance of the outer class (Outer.this),
    so the compiler automatically attaches that reference to the generated anonymous class ‚Üí it‚Äôs an anonymous inner class.

    üß© Code Examples¶

    üß± 1. Inside a static block ‚Üí Anonymous class¶

    public class Example {

    static {
        Vehicle v = new Vehicle() {
            @Override
            void start() {
                System.out.println("Starting inside static block!");
            }
        };
        v.start();
    }

    public static void main(String[] args) {
        // Class loaded ‚Üí static block runs automatically
    }
}

class Vehicle {
    void start() {
        System.out.println("Vehicle starting...");
    }
}

    Output:

    Starting inside static block!

    Explanation:

    • Created inside a static block.

    • No outer instance (Example.this does not exist).

    • So this is just an anonymous class, not an inner one.

    üß© 2. Inside a non-static method ‚Üí Anonymous inner class¶

    public class Example {

    private String garageName = "MyGarage";

    public void service() {
        Vehicle v = new Vehicle() {
            @Override
            void start() {
                System.out.println(garageName + " ‚Üí Starting after service!");
            }
        };
        v.start();
    }

    public static void main(String[] args) {
        new Example().service();
    }
}

class Vehicle {
    void start() {
        System.out.println("Vehicle starting...");
    }
}

    Output:

    MyGarage ‚Üí Starting after service!

    Explanation:

    • Created inside a non-static method.

    • Has access to the outer instance (Example.this).

    • Therefore, this is an anonymous inner class.

    • An anonymous class (in static context) will not have that reference.

    So yes ‚úÖ
